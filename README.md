# poweRcpp

> High-performance fitting of discrete power-law distributions in R via Rcpp and RcppParallel.

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

---

## Overview

**poweRcpp** provides a modern R interface to the
[powerlaw](https://github.com/alrobles/powerlaw) C++ library, a high-performance
fitter for the discrete power-law distribution.

### Attribution
This project, **poweRcpp**, is a reimplementation and extension of the original
[**DiscretePowerlawFitter**](https://github.com/CarlosManuelRodr/DiscretePowerlawFitter) library,
designed by Carlos Manuel Rodriguez. I am grateful to Carlos for his foundational work,
and this project builds on it with additional optimizations and an R interface.

Key features:

- **Maximum-likelihood estimation** of the power-law exponent α (and optional
  lower cut-off x_min) using the algorithm from Clauset, Shalizi & Newman (2009).
- **Kolmogorov-Smirnov** goodness-of-fit testing with a semi-parametric
  bootstrap, **parallelised** via [RcppParallel](https://rcppcore.github.io/RcppParallel/).
- **PDF / CDF** evaluation and **random variate generation** for discrete
  power-law distributions.
- Support for **left-bounded** (Type I) and **right-bounded** (Type II)
  distributions.
- **No external system libraries** required – the Hurwitz zeta function is
  computed in pure C++ (Euler-Maclaurin series with Bernoulli numbers).

## Quick Start

```r
library(poweRcpp)

## Simulate some power-law data
set.seed(42)
data <- powerlaw_generate(500L, alpha = 2.5, x_min = 3L)

## Fit the distribution (x_min estimated automatically)
fit <- fit_powerlaw(data)
print(fit)
#> $alpha
#> [1] 2.5
#> $x_min
#> [1] 3
#> $ks_statistic
#> [1] 0.02
#> ...

## Fit with a fixed x_min
fit2 <- fit_powerlaw(data, x_min = 3L)

## Evaluate the PMF and survival function
x <- 1L:20L
pdf_vals <- powerlaw_pdf(x, alpha = 2.5, x_min = 1L)
cdf_vals <- powerlaw_cdf(x, alpha = 2.5, x_min = 1L)

## Goodness-of-fit test (parallelised bootstrap)
gof <- powerlaw_gof(data, replicas = 1000L)
cat("p-value:", gof$p_value, "\n")
```

## Functions

| Function | Description |
|---|---|
| `fit_powerlaw(data, ...)` | Fit the power-law exponent α and (optionally) x_min by MLE + KS minimisation |
| `powerlaw_pdf(x, alpha, x_min)` | Evaluate the PMF P(X = x) |
| `powerlaw_cdf(x, alpha, x_min)` | Evaluate the survival function P(X ≥ x) |
| `powerlaw_generate(n, alpha, x_min, x_max)` | Generate n random power-law variates |
| `powerlaw_gof(data, replicas, ...)` | Bootstrap goodness-of-fit p-value (RcppParallel) |

### `fit_powerlaw`

```r
fit_powerlaw(
  data,
  x_min           = NULL,    # estimated if NULL
  alpha_precision = 0.01,    # grid step for alpha search
  type            = "left"   # "left" (Type I) or "right" (Type II)
)
```

Returns a list with:

| Field | Description |
|---|---|
| `alpha` | Estimated power-law exponent |
| `x_min` | Lower cut-off |
| `x_max` | Upper cut-off |
| `ks_statistic` | KS goodness-of-fit statistic |
| `standard_error` | Standard error of α |
| `log_likelihood` | Log-likelihood at estimated parameters |
| `n_tail` | Number of observations in the power-law tail |
| `type` | Distribution type string |
| `valid` | `TRUE` if fitting succeeded |

### `powerlaw_gof`

```r
powerlaw_gof(
  data,
  alpha    = NULL,   # estimated if NULL
  x_min    = NULL,   # estimated if NULL
  replicas = 1000L,
  type     = "left"
)
```

Returns a list with `p_value`, `ks_statistic`, `alpha`, `x_min`, and
`replicas`.  A p-value ≥ 0.10 is conventionally taken as no evidence against
the power-law hypothesis.

## Mathematical Background

A discrete variable X follows a power-law distribution if

$$P(X = x) = \frac{x^{-\alpha}}{\zeta(\alpha,\, x_{\min})}, \quad x = x_{\min}, x_{\min}+1, \ldots$$

where ζ(s, a) is the Hurwitz zeta function.  **poweRcpp** implements:

1. **Maximum-likelihood estimation** of α using **Brent's method** for
   super-linear convergence (~50 zeta evaluations regardless of
   `alpha_precision`), replacing the original fixed-step grid search that
   required O(range/precision) evaluations.
2. **x_min estimation** by minimising the KS statistic between the empirical
   and fitted survival functions (Clauset et al. 2009 §3.3).
3. **Goodness-of-fit** via a semi-parametric bootstrap: each replica is
   generated by combining a parametric power-law tail with non-parametric
   resampling of the body.

## Performance

The `powerlaw_gof` bootstrap is the most computationally expensive step.
RcppParallel distributes individual replica evaluations across all available
CPU cores using Intel TBB (or C++ std::threads as a fallback).

### Internal optimisations

| Component | Original | Improved |
|---|---|---|
| Alpha MLE | Grid search – O(range / precision) zeta calls | Brent's method – ~50 zeta calls regardless of precision |
| Empirical CDF | O((xMax − xMin) × log n) repeated binary searches | O(n + xMax − xMin) single linear pass |
| Bootstrap GOF | Single-threaded R loop | Parallel C++ loop via `RcppParallel::parallelFor` |

### Benchmark vs poweRlaw (moby dataset, R-4.x, 8-core machine)

```r
# See inst/examples/benchmark_comparison.R for the full benchmark script.
library(microbenchmark)
bench <- microbenchmark(
  poweRlaw = { m <- displ$new(moby); estimate_xmin(m) },
  poweRcpp = { fit_powerlaw(as.integer(moby)) },
  times = 5L, unit = "ms"
)
```

Typical results show **3–10× speed-up** on the full `estimate_xmin` search,
and **near-linear scaling** with core count for the bootstrap.

```r
# On a quad-core machine with 1000 replicas typically < 5 seconds
# (faster than before due to Brent's method in each bootstrap replicate)
system.time(powerlaw_gof(as.integer(moby), replicas = 1000L))
```
